**Post 1 — Announcement**

Some of the first writing I ever published was security research for IBM's SecurityIntelligence, back when I was on the X-Force team — Android malware teardowns, memory forensics, vulnerability analysis, conference recaps.

IBM has since retired that site, and the articles quietly disappeared with it. The old links now just redirect to a generic hub.

So I dug all 11 of them out of the Internet Archive and republished them on my site, with each one crediting its original IBM SecurityIntelligence run and linking back to the archived copy. The work is a decade old now, but a surprising amount of it still reads well — the fundamentals of how you take apart a piece of malware haven't changed all that much.

If you're into reverse engineering, mobile security, or memory forensics, have a look:

https://crlab.ca/learn/security-research/

**Post 2 — Threat hunting spotlight**

From the archive I just restored: a walkthrough of hunting the Rombertik malware in memory — using Volatility, CrowdInspect and AutoRuns to find injected threads and code hooks on a live host, instead of waiting for a signature to catch it.

It was written in 2015, but the proactive-threat-hunting mindset it argues for is more relevant now, not less.

https://crlab.ca/learn/security-research/proactive-threat-hunting-and-memory-forensics-against-rombertik/

**Post 3 — Mobile security spotlight**

One more from the restored archive, and still one of my favourites: how an Android device could be made to report itself fully MDM-compliant — passcode and all — without actually having a password set.

A good reminder that "the dashboard says compliant" and "the device is actually secure" are two different claims.
